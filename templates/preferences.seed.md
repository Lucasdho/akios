# akios — User Preferences

> **Seed file.** On `/akios:init` this is copied to `~/.claude/akios/preferences.md`
> (user-global, outside any repo, so it survives plugin updates). Edit the copy in
> your home directory — not this template.
>
> **What lives here:** your *transferable* coding taste — patterns you like, an
> architecture you prefer, always/never rules — that apply across **all** your
> projects. Project-specific facts (the architecture chosen for one app, a repo's
> conventions) belong in that project's native `MEMORY.md`, not here.
>
> **How it grows:** akios phases watch for preference signals — an explicit
> statement ("prefiro X") or a repeated correction (the 2nd time you undo the same
> kind of change) — and **propose** an entry before writing it. Nothing is logged
> silently; you confirm every line. Entries are deduplicated and append-only.
>
> **Priority chain** (where these preferences sit): see specs / AGENTS.md —
> `project decision → code-references → preferences (this file) → swift-dev`.
> Concrete sample code you upload outranks a stated preference here.
